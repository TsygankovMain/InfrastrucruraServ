# Правила структуры Gramax

## Как Gramax строит навигацию

Gramax читает структуру из файловой системы Git-репозитория. Навигация в левом меню = отражение папок и файлов.

**Ключевое правило:** раздел в Gramax = папка с файлом `_index.md` внутри.

---

## Структура файлов

### Раздел верхнего уровня
```
РазделНазвание/
├── _index.md          ← страница раздела (обязательно!)
├── статья-1.md        ← дочерняя статья
└── статья-2.md        ← дочерняя статья
```

### Вложенный подраздел
```
Инфраструктура/
├── _index.md          ← страница раздела "Инфраструктура"
├── servers/
│   ├── _index.md      ← страница подраздела "Серверы"
│   ├── pvd-1c.md      ← статья о сервере
│   └── другой-сервер.md
└── network/
    ├── _index.md      ← страница подраздела "Сеть"
    └── ip-allocation.md
```

### ❌ Так НЕ работает
```
# Gramax игнорирует index.md (без подчёркивания)!
Обзор/
└── index.md           ← НЕ создаёт раздел, будет плоским списком

# Также НЕ работает файл снаружи папки
Обзор.md               ← Gramax не связывает с папкой Обзор/
Обзор/
└── статья.md
```

## Frontmatter обязательные поля

Каждый `.md` файл должен иметь frontmatter:

```yaml
---
title: "Название статьи на русском"
order: 1
---
```

| Поле | Описание |
|------|----------|
| `title` | Название в навигации. **Только русский язык.** |
| `order` | Порядковый номер в меню (1 = первый, 99 = в конце) |

### Порядок разделов (order в _index.md)
| Раздел | order |
|--------|-------|
| Обзор | 1 |
| Инфраструктура | 2 |
| Сервисы | 3 |
| Доступ | 4 |
| Ранбуки | 5 |
| Инструкции AI | 99 |

## Как добавить новую статью

1. Создать файл `НазваниеСтатьи.md` в нужной папке раздела
2. Добавить frontmatter с `title` и `order`
3. Написать контент начиная с `# Заголовок`
4. Сделать git push
5. Синхронизировать каталог в Gramax (кнопка ↕ внизу слева)

**Пример:** добавляем статью о новом сервере:
```
Инфраструктура/servers/new-server.md
```
```markdown
---
title: "NEW-SERVER: Описание сервера"
order: 3
---

# NEW-SERVER: Описание сервера
...
```

---

## Как добавить новый раздел

1. Создать папку с названием на **русском языке**
2. Создать внутри `_index.md` с frontmatter
3. Добавить статьи в папку
4. Git push + синхронизация

**Пример:** новый раздел "Мониторинг инцидентов":
```
Инциденты/
├── _index.md       ← title: "Инциденты", order: 6
├── log-analysis.md
└── escalation.md
```

## Что делать НЕЛЬЗЯ

| ❌ Нельзя | ✅ Нужно |
|-----------|---------|
| Называть файл разделa `index.md` | Называть `_index.md` |
| Использовать английские названия в `title` | Только русский |
| Создавать раздел без `_index.md` | Всегда создавать `_index.md` |
| Смешивать языки в одном заголовке | Выбрать один язык |

## Что значит "Pending" в Gramax

**Pending** (ожидающие) — это изменения, которые вы сделали локально в Gramax, но ещё **не опубликовали в Git**.

```
[Pending]  = изменено локально, не сохранено в репозиторий
```

### Жизненный цикл изменения:

```
Редактирование → Pending → Commit → Push → Опубликовано
```

| Статус | Что значит |
|--------|------------|
| **Pending** | Статья изменена, но НЕ запушена в Git. Видна только вам |
| **Published** | Изменение в Git. Видно всем после синхронизации портала |

### Как убрать Pending:

В Gramax приложении:
1. Нажать кнопку **Git** (иконка ветки внизу слева)
2. Написать комментарий к изменению
3. Нажать **Commit & Push**

Или через терминал:
```bash
git add -A
git commit -m "Описание изменений"
git push
```

> **Важно:** если у вас много pending-изменений и вы синхронизируете из Git — локальные изменения могут конфликтовать с внешними. Делайте commit перед синхронизацией.
